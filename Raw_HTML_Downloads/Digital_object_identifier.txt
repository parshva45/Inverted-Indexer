  <p>\n       In computing, a\n       <b>\n        Digital Object Identifier\n       </b>\n       or\n       <b>\n        DOI\n       </b>\n       is a\n       <a href="/wiki/Persistent_identifier" title="Persistent identifier">\n        persistent identifier\n       </a>\n       or\n       <a href="/wiki/Handle_(computing)" title="Handle (computing)">\n        handle\n       </a>\n       used to uniquely identify objects, standardized by the International Organization for Standardization (\n       <a class="mw-redirect" href="/wiki/ISO" title="ISO">\n        ISO\n       </a>\n       ).\n       <sup class="reference" id="cite_ref-iso_1-0">\n        <a href="#cite_note-iso-1">\n         [1]\n        </a>\n       </sup>\n       An implementation of the\n       <a href="/wiki/Handle_System" title="Handle System">\n        Handle System\n       </a>\n       ,\n       <sup class="reference" id="cite_ref-2">\n        <a href="#cite_note-2">\n         [2]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-3">\n        <a href="#cite_note-3">\n         [3]\n        </a>\n       </sup>\n       DOIs are in wide use mainly to identify academic, professional, and government information, such as journal articles, research reports and data sets, and official publications though they also have been used to identify other types of information resources, such as commercial videos.\n      </p>\n      <p>\n       A DOI aims to be "resolvable", usually to some form of access to the information object to which the DOI refers. This is achieved by binding the DOI to\n       <a href="/wiki/Metadata" title="Metadata">\n        metadata\n       </a>\n       about the object, such as a\n       <a href="/wiki/URL" title="URL">\n        URL\n       </a>\n       , indicating where the object can be found. Thus, by being actionable and interoperable, a DOI differs from identifiers such as\n       <a class="mw-redirect" href="/wiki/ISBN" title="ISBN">\n        ISBNs\n       </a>\n       and\n       <a class="mw-redirect" href="/wiki/ISRC" title="ISRC">\n        ISRCs\n       </a>\n       which aim only to uniquely identify their referents. The DOI system uses the\n       <a href="/wiki/Indecs_Content_Model" title="Indecs Content Model">\n        indecs Content Model\n       </a>\n       for representing metadata.\n      </p>\n      <p>\n       The DOI for a document remains fixed over the lifetime of the document, whereas its location and other metadata may change. Referring to an online document by its DOI shall provide a more stable linking than simply using its URL. Every time a URL changes, the publisher has to update the metadata for the DOI to link to the new URL.\n       <sup class="reference" id="cite_ref-4">\n        <a href="#cite_note-4">\n         [4]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-5">\n        <a href="#cite_note-5">\n         [5]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-6">\n        <a href="#cite_note-6">\n         [6]\n        </a>\n       </sup>\n       It is the publisher\'s responsibility to update the DOI database. If he fails to do so, the DOI resolves to a\n       <a href="/wiki/Link_rot" title="Link rot">\n        dead link\n       </a>\n       leaving the DOI useless.\n      </p>\n      <p>\n       The developer and administrator of the DOI system is the International DOI Foundation (IDF), which introduced it in 2000.\n       <sup class="reference" id="cite_ref-7">\n        <a href="#cite_note-7">\n         [7]\n        </a>\n       </sup>\n       Organizations that meet the contractual obligations of the DOI system and are willing to pay to become a member of the system can assign DOIs.\n       <sup class="reference" id="cite_ref-dd_8-0">\n        <a href="#cite_note-dd-8">\n         [8]\n        </a>\n       </sup>\n       The DOI system is implemented through a federation of registration agencies coordinated by the IDF.\n       <sup class="reference" id="cite_ref-9">\n        <a href="#cite_note-9">\n         [9]\n        </a>\n       </sup>\n       By late April 2011 more than 50 million DOI names had been assigned by some 4,000 organizations,\n       <sup class="reference" id="cite_ref-10">\n        <a href="#cite_note-10">\n         [10]\n        </a>\n       </sup>\n       and by April 2013 this number had grown to 85 million DOI names assigned through 9,500 organizations.\n      </p>\n      <p>\n      </p>\n      <div class="toc" id="toc">\n       <div </li>\n       </ul>\n      </div>\n      <p>\n      </p>\n      <h2>\n       <span class="mw-headline" id="Nomenclature">\n        Nomenclature\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=1" title="Edit section: Nomenclature">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       A DOI is a type of Handle System handle, which takes the form of a\n       <a class="mw-redirect" href="/wiki/Character_string" title="Character string">\n        character string\n       </a>\n       divided into two parts, a prefix and a suffix, separated by a slash. The prefix identifies the registrant of the identifier, and the suffix is chosen by the registrant and identifies the specific object associated with that DOI. Most legal\n       <a href="/wiki/Unicode" title="Unicode">\n        Unicode\n       </a>\n       characters are allowed in these strings, which are interpreted in a\n       <a class="mw-redirect" href="/wiki/Case-insensitive" title="Case-insensitive">\n        case-insensitive\n       </a>\n       manner. The prefix usually takes the form\n       <code>\n        10.NNNN\n       </code>\n       , where\n       <code>\n        NNNN\n       </code>\n       is a series of at least 4 numbers greater than or equal to\n       <code>\n        1000\n       </code>\n       , whose limit depends only on the total number of registrants.\n       <sup class="reference" id="cite_ref-CrossRefDOI_11-0">\n        <a href="#cite_note-CrossRefDOI-11">\n         [11]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-DOIKeyFacts_12-0">\n        <a href="#cite_note-DOIKeyFacts-12">\n         [12]\n        </a>\n       </sup>\n       The prefix may be further subdivided with periods, like\n       <code>\n        10.NNNN.N\n       </code>\n       .\n       <sup class="reference" id="cite_ref-2.2.2_13-0">\n        <a href="#cite_note-2.2.2-13">\n         [13]\n        </a>\n       </sup>\n      </p>\n      <p>\n       For example, in the DOI name\n       <code>\n        10.1000/182\n       </code>\n       , the prefix is\n       <code>\n        10.1000\n       </code>\n       and the suffix is\n       <code>\n        182\n       </code>\n       . The "10." part of the prefix distinguishes the handle as part of the DOI namespace, as opposed to some other Handle System namespace,\n       <sup class="reference" id="cite_ref-14">\n        <a href="#cite_note-14">\n         [A]\n        </a>\n       </sup>\n       and the characters\n       <code>\n        1000\n       </code>\n       in the prefix identify the registrant; in this case the registrant is the International DOI Foundation itself.\n       <code>\n        182\n       </code>\n       is the suffix, or item ID, identifying a single object (in this case, the latest version of the\n       <i>\n        DOI Handbook\n       </i>\n       ).\n      </p>\n      <p>\n       DOI names can identify creative works (such as texts, images, audio or video items, and software) in both electronic and physical forms,\n       <a href="/wiki/Performance" title="Performance">\n        performances\n       </a>\n       , and abstract works\n       <sup class="reference" id="cite_ref-doifaq2_15-0">\n        <a href="#cite_note-doifaq2-15">\n         [14]\n        </a>\n       </sup>\n       such as licenses, parties to a transaction, etc.\n      </p>\n      <p>\n       The names can refer to objects at varying levels of detail: thus DOI names can identify a journal, an individual issue of a journal, an individual article in the journal, or a single table in that article. The choice of level of detail is left to the assigner, but in the DOI system it must be declared as part of the metadata that is associated with a DOI name, using a data dictionary based on the\n       <a href="/wiki/Indecs_Content_Model" title="Indecs Content Model">\n        indecs Content Model\n       </a>\n       .\n      </p>\n      <h3>\n       <span class="mw-headline" id="Display">\n        Display\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=2" title="Edit section: Display">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h3>\n      <p>\n       The official\n       <i>\n        DOI Handbook\n       </i>\n       explicitly states that DOIs should display on screens and in print in the format "doi:10.1000/182".\n       <sup class="reference" id="cite_ref-C4WDefault-2811140_16-0">\n        <a href="#cite_note-C4WDefault-2811140-16">\n         [15]\n        </a>\n       </sup>\n       Contrary to the\n       <i>\n        DOI Handbook\n       </i>\n       ,\n       <a class="mw-redirect" href="/wiki/CrossRef" title="CrossRef">\n        CrossRef\n       </a>\n       , a major DOI registration agency, recommends displaying a URL (for example,\n       <code>\n        https://doi.org/10.1000/182\n       </code>\n       ) instead of the officially specified format (for example,\n       <code>\n        <a class="external text" href="https://doi.org/10.1000/182" rel="nofollow">\n         doi:10.1000/182\n        </a>\n       </code>\n       )\n       <sup class="reference" id="cite_ref-17">\n        <a href="#cite_note-17">\n         [16]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-18">\n        <a href="#cite_note-18">\n         [17]\n        </a>\n       </sup>\n       This URL provides the location of an\n       <a class="mw-redirect" href="/wiki/HTTP_proxy" title="HTTP proxy">\n        HTTP proxy\n       </a>\n       server which will redirect web accesses to the correct online location of the linked item.\n       <sup class="reference" id="cite_ref-dd_8-1">\n        <a href="#cite_note-dd-8">\n         [8]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-19">\n        <a href="#cite_note-19">\n         [18]\n        </a>\n       </sup>\n       This recommendation is primarily based on the assumption that the DOI is being displayed without being hyper-linked to its appropriate URL \xe2\x80\x93 the argument being that without the hyperlink it is not as easy to copy-and-paste the full URL to actually bring up the page for the DOI, thus the entire URL should be displayed, allowing people viewing the page containing the DOI to copy-and-paste the URL, by hand, into a new window/tab in their\n       <a href="/wiki/Web_browser" title="Web browser">\n        browser\n       </a>\n       in order to go to the appropriate page for the document the DOI represents.\n      </p>\n      <h2>\n       <span class="mw-headline" id="Applications">\n        Applications\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=3" title="Edit section: Applications">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       Major applications of the DOI system currently include:\n      </p>\n      <ul>\n       <li>\n        <a href="/wiki/Scientific_literature" title="Scientific literature">\n         scholarly materials\n        </a>\n        (journal articles, books, ebooks,etc.) through\n        <a class="mw-redirect" href="/wiki/CrossRef" title="CrossRef">\n         CrossRef\n        </a>\n        , a consortium of around 3,000 publishers;\n        <a href="/wiki/Airiti" title="Airiti">\n         Airiti\n        </a>\n        , a leading provider of electronic academic journals in Chinese and Taiwanese; and the Japan Link Center (JaLC) an organization providing link management and DOI assignment for electronic academic journals in Japanese.\n       </li>\n       <li>\n        research datasets through\n        <a href="/wiki/DataCite" title="DataCite">\n         DataCite\n        </a>\n        , a consortium of leading research libraries, technical information providers, and scientific data centers;\n       </li>\n       <li>\n        <a href="/wiki/European_Union" title="European Union">\n         European Union\n        </a>\n        official publications through the\n        <a class="mw-redirect" href="/wiki/Publications_Office_(European_Union)" title="Publications Office (European Union)">\n         EU publications office\n        </a>\n        ;\n       </li>\n       <li>\n        the\n        <a href="/wiki/CNKI" title="CNKI">\n         Chinese National Knowledge Infrastructure\n        </a>\n        project at\n        <a href="/wiki/Tsinghua_University" title="Tsinghua University">\n         Tsinghua University\n        </a>\n        and the Institute of Scientific and Technical Information of China (ISTIC), two initiatives sponsored by the Chinese government.\n       </li>\n       <li>\n        Permanent global identifiers for commercial video content through the Entertainment ID Registry, commonly known as\n        <a href="/wiki/EIDR" title="EIDR">\n         EIDR\n        </a>\n        .\n       </li>\n      </ul>\n      <p>\n       In the\n       <a class="mw-redirect" href="/wiki/Organisation_for_Economic_Co-operation_and_Development" title="Organisation for Economic Co-operation and Development">\n        Organisation for Economic Co-operation and Development\n       </a>\n       \'s publication service\n       <a href="/wiki/OECD_iLibrary" title="OECD iLibrary">\n        OECD iLibrary\n       </a>\n       , each table or graph in an OECD publication is shown with a DOI name that leads to an Excel file of data underlying the tables and graphs. Further development of such services is planned.\n       <sup class="reference" id="cite_ref-20">\n        <a href="#cite_note-20">\n         [19]\n        </a>\n       </sup>\n      </p>\n      <p>\n       A multilingual European DOI registration agency activity,\n       <a class="external text" href="http://www.mEDRA.org" rel="nofollow">\n        <i>\n         m\n        </i>\n        EDRA\n       </a>\n       , and a Chinese registration agency,\n       <a class="external text" href="http://www.wanfangdata.com/" rel="nofollow">\n        Wanfang Data\n       </a>\n       , are active in non-English language markets.\n      </p>\n      <h2>\n       <span class="mw-headline" id="Features_and_benefits">\n        Features and benefits\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=4" title="Edit section: Features and benefits">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       The IDF designed the DOI system to provide a form of\n       <a href="/wiki/Persistent_identifier" title="Persistent identifier">\n        persistent identification\n       </a>\n       , in which each DOI name permanently and unambiguously identifies the object to which it is associated. It also associates\n       <a href="/wiki/Metadata" title="Metadata">\n        metadata\n       </a>\n       with objects, allowing it to provide users with relevant pieces of information about the objects and their relationships. Included as part of this metadata are network actions that allow DOI names to be resolved to web locations where the objects they describe can be found. To achieve its goals, the DOI system combines the\n       <a href="/wiki/Handle_System" title="Handle System">\n        Handle System\n       </a>\n       and the\n       <a href="/wiki/Indecs_Content_Model" title="Indecs Content Model">\n        indecs Content Model\n       </a>\n       with a social infrastructure.\n      </p>\n      <p>\n       The Handle System ensures that the DOI name for an object is not based on any changeable attributes of the object such as its physical location or ownership, that the attributes of the object are encoded in its metadata rather than in its DOI name, and that no two objects are assigned the same DOI name. Because DOI names are short character strings, they are human-readable, may be copied and pasted as text, and fit into the\n       <a href="/wiki/Uniform_Resource_Identifier" title="Uniform Resource Identifier">\n        URI\n       </a>\n       specification. The DOI name-resolution mechanism acts behind the scenes, so that users communicate with it in the same way as with any other web service; it is built on\n       <a href="/wiki/Open_architecture" title="Open architecture">\n        open architectures\n       </a>\n       , incorporates\n       <a href="/wiki/Computational_trust" title="Computational trust">\n        trust mechanisms\n       </a>\n       , and is engineered to operate reliably and flexibly so that it can be adapted to changing demands and new applications of the DOI system.\n       <sup class="reference" id="cite_ref-21">\n        <a href="#cite_note-21">\n         [20]\n        </a>\n       </sup>\n       DOI name-resolution may be used with\n       <a href="/wiki/OpenURL" title="OpenURL">\n        OpenURL\n       </a>\n       to select the most appropriate among multiple locations for a given object, according to the location of the user making the request.\n       <sup class="reference" id="cite_ref-22">\n        <a href="#cite_note-22">\n         [21]\n        </a>\n       </sup>\n       However, despite this ability, the DOI system has drawn criticism from librarians for directing users to non-free copies of documents that would have been available for no additional fee from alternative locations.\n       <sup class="reference" id="cite_ref-23">\n        <a href="#cite_note-23">\n         [22]\n        </a>\n       </sup>\n      </p>\n      <p>\n       The\n       <a href="/wiki/Indecs_Content_Model" title="Indecs Content Model">\n        indecs Content Model\n       </a>\n       as used within the DOI system associates metadata with objects. A small kernel of common metadata is shared by all DOI names and can be optionally extended with other relevant data, which may be public or restricted. Registrants may update the metadata for their DOI names at any time, such as when publication information changes or when an object moves to a different URL.\n      </p>\n      <p>\n       The International DOI Foundation (IDF) oversees the integration of these technologies and operation of the system through a technical and social infrastructure. The social infrastructure of a federation of independent registration agencies offering DOI services was modelled on existing successful federated deployments of identifiers such as\n       <a href="/wiki/GS1" title="GS1">\n        GS1\n       </a>\n       and\n       <a class="mw-redirect" href="/wiki/ISBN" title="ISBN">\n        ISBN\n       </a>\n       .\n      </p>\n      <h2>\n       <span class="mw-headline" id="Comparison_with_other_identifier_schemes">\n        Comparison with other identifier schemes\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=5" title="Edit section: Comparison with other identifier schemes">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       A DOI name differs from commonly used Internet pointers to material, such as the\n       <a class="mw-redirect" href="/wiki/Uniform_Resource_Locator" title="Uniform Resource Locator">\n        Uniform Resource Locator\n       </a>\n       (URL), in that it identifies an object itself as a\n       <a href="/wiki/First_class_(computing)" title="First class (computing)">\n        first-class entity\n       </a>\n       , rather than the specific place where the object is located at a certain time. It implements the\n       <a href="/wiki/Uniform_Resource_Identifier" title="Uniform Resource Identifier">\n        Uniform Resource Identifier\n       </a>\n       (\n       <a href="/wiki/Uniform_Resource_Name" title="Uniform Resource Name">\n        Uniform Resource Name\n       </a>\n       ) concept and adds to it a data model and social infrastructure.\n       <sup class="reference" id="cite_ref-24">\n        <a href="#cite_note-24">\n         [23]\n        </a>\n       </sup>\n      </p>\n      <p>\n       A DOI name also differs from standard identifier registries such as the\n       <a href="/wiki/International_Standard_Book_Number" title="International Standard Book Number">\n        ISBN\n       </a>\n       ,\n       <a href="/wiki/International_Standard_Recording_Code" title="International Standard Recording Code">\n        ISRC\n       </a>\n       , etc. The purpose of an identifier registry is to manage a given collection of identifiers, whereas the primary purpose of the DOI system is to make a collection of identifiers actionable and interoperable, where that collection can include identifiers from many other controlled collections.\n       <sup class="reference" id="cite_ref-25">\n        <a href="#cite_note-25">\n         [24]\n        </a>\n       </sup>\n      </p>\n      <p>\n       The DOI system offers persistent,\n       <a href="/wiki/Semantic_interoperability" title="Semantic interoperability">\n        semantically-interoperable\n       </a>\n       resolution to related current data and is best suited to material that will be used in services outside the direct control of the issuing assigner (e.g., public citation or managing content of value). It uses a managed registry (providing social and technical infrastructure). It does not assume any specific business model for the provision of identifiers or services and enables other existing services to link to it in defined ways. Several approaches for making identifiers persistent have been proposed. The comparison of persistent identifier approaches is difficult because they are not all doing the same thing. Imprecisely referring to a set of schemes as "identifiers" doesn\'t mean that they can be compared easily. Other "identifier systems" may be enabling technologies with low barriers to entry, providing an easy to use labeling mechanism that allows anyone to set up a new instance (examples include\n       <a class="mw-redirect" href="/wiki/Persistent_Uniform_Resource_Locator" title="Persistent Uniform Resource Locator">\n        Persistent Uniform Resource Locator\n       </a>\n       (PURL), URLs,\n       <a class="mw-redirect" href="/wiki/Globally_Unique_Identifier" title="Globally Unique Identifier">\n        Globally Unique Identifiers\n       </a>\n       (GUIDs), etc.), but may lack some of the functionality of a registry-controlled scheme and will usually lack accompanying metadata in a controlled scheme. The DOI system does not have this approach and should not be compared directly to such identifier schemes. Various applications using such enabling technologies with added features have been devised that meet some of the features offered by the DOI system for specific sectors (e.g.,\n       <a href="/wiki/Archival_Resource_Key" title="Archival Resource Key">\n        ARK\n       </a>\n       ).\n      </p>\n      <p>\n       A DOI name does not depend on the object\'s location and, in this way, is similar to a Uniform Resource Name (URN) or PURL but differs from an ordinary URL. URLs are often used as substitute identifiers for documents on the Internet (better characterised as Uniform Resource Identifiers) although the same document at two different locations has two URLs. By contrast, persistent identifiers such as DOI names identify objects as first class entities: two instances of the same object would have the same DOI name.\n      </p>\n      <h2>\n       <span class="mw-headline" id="Resolution">\n        Resolution\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=6" title="Edit section: Resolution">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       DOI name resolution is provided through the\n       <a href="/wiki/Handle_System" title="Handle System">\n        Handle System\n       </a>\n       , developed by\n       <a href="/wiki/Corporation_for_National_Research_Initiatives" title="Corporation for National Research Initiatives">\n        Corporation for National Research Initiatives\n       </a>\n       , and is freely available to any user encountering a DOI name. Resolution redirects the user from a DOI name to one or more pieces of typed data: URLs representing instances of the object, services such as e-mail, or one or more items of metadata. To the Handle System, a DOI name is a handle, and so has a set of values assigned to it and may be thought of as a record that consists of a group of fields. Each handle value must have a data type specified in its\n       <code>\n        &lt;type&gt;\n       </code>\n       field, which defines the syntax and semantics of its data. While a DOI persistently and uniquely identifies the object to which it is assigned, DOI resolution may not be persistent, due to technical and administrative issues.\n      </p>\n      <p>\n       To resolve a DOI name, it may be input to a DOI resolver, such as\n       <a class="external text" href="https://doi.org/" rel="nofollow">\n        doi.org\n       </a>\n       .\n      </p>\n      <p>\n       Another approach, which avoids typing or\n       <a class="mw-redirect" href="/wiki/Cut-and-paste" title="Cut-and-paste">\n        cutting-and-pasting\n       </a>\n       into a resolver is to include the DOI in a document as a URL which uses the resolver as an HTTP proxy, such as\n       <code>\n        http://doi.org/\n       </code>\n       (preferred)\n       <sup class="reference" id="cite_ref-26">\n        <a href="#cite_note-26">\n         [25]\n        </a>\n       </sup>\n       or\n       <code>\n        http://dx.doi.org/\n       </code>\n       , both of which support HTTPS. For example, the DOI\n       <code>\n        10.1000/182\n       </code>\n       can be included in a reference or\n       <a href="/wiki/Hyperlink" title="Hyperlink">\n        hyperlink\n       </a>\n       as\n       <code>\n        <a class="external free" href="https://doi.org/10.1000/182" rel="nofollow">\n         https://doi.org/10.1000/182\n        </a>\n       </code>\n       . This approach allows users to click on the DOI as a normal\n       <a href="/wiki/Hyperlink" title="Hyperlink">\n        hyperlink\n       </a>\n       . Indeed, as previously mentioned, this is how CrossRef recommends that DOIs always be represented (preferring HTTPS over HTTP), so that if they are cut-and-pasted into other documents, emails, etc., they will be actionable.\n      </p>\n      <p>\n       Other DOI resolvers and HTTP Proxies include\n       <a class="external free" href="http://hdl.handle.net" rel="nofollow">\n        http://hdl.handle.net\n       </a>\n       ,\n       <a class="external free" href="http://doi.medra.org" rel="nofollow">\n        http://doi.medra.org\n       </a>\n       ,\n       <a class="external free" href="https://doi.pangaea.de/" rel="nofollow">\n        https://doi.pangaea.de/\n       </a>\n       . At the beginning of the year 2016, a new class of alternative DOI resolvers was started by\n       <a class="external free" href="http://doai.io" rel="nofollow">\n        http://doai.io\n       </a>\n       . This service is unusual in that it tries to find a\n       <a href="/wiki/Paywall" title="Paywall">\n        non-paywalled\n       </a>\n       version of a title and redirects you to that instead of the publisher\'s version.\n       <sup class="reference" id="cite_ref-capsh_27-0">\n        <a href="#cite_note-capsh-27">\n         [26]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-28">\n        <a href="#cite_note-28">\n         [27]\n        </a>\n       </sup>\n       Since then, other open-access favoring DOI resolvers have been created, notably\n       <a class="external free" href="https://oadoi.org/" rel="nofollow">\n        https://oadoi.org/\n       </a>\n       in October 2016.\n       <sup class="reference" id="cite_ref-impactstory_29-0">\n        <a href="#cite_note-impactstory-29">\n         [28]\n        </a>\n       </sup>\n       While traditional DOI resolvers solely rely on the Handle System, alternative DOI resolvers first consult open access resources such as\n       <a href="/wiki/BASE_(search_engine)" title="BASE (search engine)">\n        BASE\n       </a>\n       (Bielefeld Academic Search Engine).\n       <sup class="reference" id="cite_ref-capsh_27-1">\n        <a href="#cite_note-capsh-27">\n         [26]\n        </a>\n       </sup>\n       <sup class="reference" id="cite_ref-impactstory_29-1">\n        <a href="#cite_note-impactstory-29">\n         [28]\n        </a>\n       </sup>\n      </p>\n      <p>\n       An alternative to HTTP proxies is to use one of a number of add-ons and plug-ins for\n       <a href="/wiki/Web_browser" title="Web browser">\n        browsers\n       </a>\n       , thereby avoiding the conversion of the DOIs to URLs,\n       <sup class="reference" id="cite_ref-30">\n        <a href="#cite_note-30">\n         [29]\n        </a>\n       </sup>\n       which depend on domain names and may be subject to change, while still allowing the DOI to be treated as a normal hyperlink. For example. the\n       <a class="external text" href="https://handle.net/tools/extensions/firefox_hdlclient.html" rel="nofollow">\n        CNRI Handle Extension for Firefox\n       </a>\n       , enables the browser to access Handle System handles or DOIs like\n       <code>\n        hdl:4263537/4000\n       </code>\n       or\n       <code>\n        doi:10.1000/1\n       </code>\n       directly in the\n       <a href="/wiki/Firefox" title="Firefox">\n        Firefox\n       </a>\n       browser, using the native Handle System protocol. This plug-in can also replace references to web-to-handle proxy servers with native resolution. A disadvantage of this approach for publishers is that, at least at present, most users will be encountering the DOIs in a browser,\n       <a class="mw-redirect" href="/wiki/Mail_reader" title="Mail reader">\n        mail reader\n       </a>\n       , or other software which does not have one of these plug-ins installed.\n      </p>\n      <h2>\n       <span class="mw-headline" id="IDF_organizational_structure">\n        IDF organizational structure\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=7" title="Edit section: IDF organizational structure">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       The International DOI Foundation (IDF), a non-profit organisation created in 1998, is the governance body of the DOI system.\n       <sup class="reference" id="cite_ref-31">\n        <a href="#cite_note-31">\n         [30]\n        </a>\n       </sup>\n       It safeguards all\n       <a href="/wiki/Intellectual_property" title="Intellectual property">\n        intellectual property rights\n       </a>\n       relating to the DOI system, manages common operational features, and supports the development and promotion of the DOI system. The IDF ensures that any improvements made to the DOI system (including creation, maintenance, registration, resolution and policymaking of DOI names) are available to any DOI registrant. It also prevents third parties from imposing additional licensing requirements beyond those of the IDF on users of the DOI system.\n      </p>\n      <p>\n       The IDF is controlled by a Board elected by the members of the Foundation, with an appointed Managing Agent who is responsible for co-ordinating and planning its activities. Membership is open to all organizations with an interest in electronic publishing and related enabling technologies. The IDF holds annual open meetings on the topics of DOI and related issues.\n      </p>\n      <p>\n       Registration agencies, appointed by the IDF, provide services to DOI registrants: they allocate DOI prefixes, register DOI names, and provide the necessary infrastructure to allow registrants to declare and maintain metadata and state data. Registration agencies are also expected to actively promote the widespread adoption of the DOI system, to cooperate with the IDF in the development of the DOI system as a whole, and to provide services on behalf of their specific user community. A list of current RAs is maintained by the International DOI Foundation. The IDF is recognized as one of the federated registrars for the Handle System by the DONA Foundation (of which the IDF is a board member), and is responsible for assigning Handle System prefixes under the top-level\n       <code>\n        10\n       </code>\n       prefix.\n       <sup class="reference" id="cite_ref-32">\n        <a href="#cite_note-32">\n         [31]\n        </a>\n       </sup>\n      </p>\n      <p>\n       Registration agencies generally charge a fee to assign a new DOI name; parts of these fees are used to support the IDF. The DOI system overall, through the IDF, operates on a\n       <a class="mw-redirect" href="/wiki/Not-for-profit" title="Not-for-profit">\n        not-for-profit\n       </a>\n       cost recovery basis.\n      </p>\n      <h2>\n       <span class="mw-headline" id="Standardization">\n        Standardization\n       </span>\n       <span class="mw-editsection">\n        <span class="mw-editsection-bracket">\n         [\n        </span>\n        <a href="/w/index.php?title=Digital_object_identifier&amp;action=edit&amp;section=8" title="Edit section: Standardization">\n         edit\n        </a>\n        <span class="mw-editsection-bracket">\n         ]\n        </span>\n       </span>\n      </h2>\n      <p>\n       The DOI system is an international standard developed by the\n       <a href="/wiki/International_Organization_for_Standardization" title="International Organization for Standardization">\n        International Organization for Standardization\n       </a>\n       in its technical committee on identification and description, TC46/SC9.\n       <sup class="reference" id="cite_ref-33">\n        <a href="#cite_note-33">\n         [32]\n        </a>\n       </sup>\n       The Draft International Standard ISO/DIS 26324, Information and documentation \xe2\x80\x93 Digital Object Identifier System met the ISO requirements for approval. The relevant ISO Working Group later submitted an edited version to ISO for distribution as an FDIS (Final Draft International Standard) ballot,\n       <sup class="reference" id="cite_ref-34">\n        <a href="#cite_note-34">\n         [33]\n        </a>\n       </sup>\n       which was approved by 100% of those voting in a ballot closing on 15 November 2010.\n       <sup class="reference" id="cite_ref-35">\n        <a href="#cite_note-35">\n         [34]\n        </a>\n       </sup>\n       The final standard was published on 23 April 2012.\n       <sup class="reference" id="cite_ref-iso_1-1">\n        <a href="#cite_note-iso-1">\n         [1]\n        </a>\n       </sup>\n      </p>\n      <p>\n       DOI is a registered URI under the\n       <a href="/wiki/Info_URI_scheme" title="Info URI scheme">\n        info URI scheme\n       </a>\n       specified by IETF\n       <a class="external mw-magiclink-rfc" href="//tools.ietf.org/html/rfc4452" rel="nofollow">\n        RFC 4452\n       </a>\n       . info:doi/ is the infoURI Namespace of Digital Object Identifiers.\n       <sup class="reference" id="cite_ref-36">\n        <a href="#cite_note-36">\n         [35]\n        </a>\n       </sup>\n      </p>\n      <p>\n       The DOI syntax is a\n       <a class="mw-redirect" href="/wiki/NISO" title="NISO">\n        NISO\n       </a>\n       standard, first standardised in 2000, ANSI/NISO Z39.84-2005 Syntax for the Digital Object Identifier.\n       <sup class="reference" id="cite_ref-37">\n        <a href="#cite_note-37">\n         [36]\n        </a>\n       </sup>\n      </p>\n      <p>\n       The maintainers of the DOI system have deliberately not registered a DOI namespace for URNs, stating that:\n      </p>\n      <blockquote class="templatequote">\n       <p>\n        URN architecture assumes a DNS-based Resolution Discovery Service (RDS) to find the service appropriate to the given URN scheme. However no such widely deployed RDS schemes currently exist.... DOI is not registered as a URN namespace, despite fulfilling all the functional requirements, since URN registration appears to offer no advantage to the DOI System. It requires an additional layer of administration for defining DOI as a URN namespace (the string\n        <code class="mw-highlight" dir="ltr" id="" style="">\n         urn:doi:10.1000/1\n        </code>\n        rather than the simpler\n        <code class="mw-highlight" dir="ltr" id="" style="">\n         doi:10.1000/1\n        </code>\n        ) and an additional step of unnecessary redirection to access the resolution service, already achieved through either http proxy or native resolution. If RDS mechanisms supporting URN specifications become widely available, DOI will be registered as a URN.\n       </p>\n       <div class="templatequotecite">\n        \xe2\x80\x94\n        <cite>\n
